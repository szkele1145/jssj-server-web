# 建设世界服务器官网 — 项目上下文

## 项目简介
Minecraft 服务器"建设世界"的官网，含服务器状态、公告、投票、封禁榜、神人榜等社区功能。

## 技术栈
- 纯静态 HTML + CSS + JS，无框架
- 后端：Cloudflare Workers + KV（`api.jssj.cc.cd`）
- 图标：Font Awesome 6
- 部署：GitHub Pages（`jssj.cc.cd` → CNAME）

## 关键决策
- 投票用 IP 防刷（`CF-Connecting-IP`），前端 localStorage 辅助
- 投票 Worker 不做去重计数 → 改由 IP + KV voters 追踪
- 图片链接处理从 admin 前端移到 Worker 后端，管理员只管贴链接
- 不引入依赖/框架，纯静态维护

## 已知问题/Bug
- Worker 内 fetch mcsrvstat.us 偶发 403/超时（CF IP 被限），现已 fallback v3+v2 双版本 + 3 次重试
- CF KV 最终一致性，并发投票可能覆盖（低概率，可接受）
- 浏览器直连 mcsrvstat.us 降级可能因 CORS 失败（fallback 逻辑，可接受）

## 神人榜图片方案改进（2026-07-24）
- **问题**：后台把文本和图片链接混在一个字段，靠扩展名自动检测转 `<img>`，下载链接也被强行转换，方案不可靠
- **方案**：后台表单拆为「事件描述」+「图片链接」两个独立字段，Worker 分开存储，响应时组装 HTML
- **兼容**：旧数据（`content` 字段）保持原样；新数据（`text`+`images`）动态构建 `content`
- `admin.html`：`storyContent` → `storyText` + `storyImageUrls`
- `worker.js`：POST 存储 `text`/`images`，GET 动态构建 `content`（所有图片链接渲染为 `<img>`，先试直链，失败自动 fallback 到 Worker 代理 `/api/img`）
- `worker.js`：`/api/img` 代理改进（10s 超时、强制 `image/jpeg`、命中失败返回 SVG 占位图）
